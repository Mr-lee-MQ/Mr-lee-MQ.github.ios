Zookeeper

1、Zookeeper入门

  • Dubbo框架 SpringCloud框架 充当注册中心的角色
  • Hadoop Hbase 组件 集群架构 zk集群管理者
  • zk实现分布式锁 redis分布式

1.1、Zookeeper的工作机制

​ Zookeeper从设计模式角度来理解：是一个基于观察者模式设计的分布式服务管理框架，他负责存储和管理大家都关心的数据，然后接受观察者的注册，一旦这些数据的状态发生变化，Zookeeper就会负责通知已经在Zookeeper上注册的那些观察者作出相应的反应。

1.2、Zookeeper特点

  1. Zookeeper：一个领导者（Leader），多个跟随者（Follower）组成的集群。
  2. 集群中只要有半数以上的节点存活，Zookeeper集群就可以正常服务。
  3. 全局数据一致：每个Server保存相同的数据副本，Client无论连接到哪个Server，数据都是一致的。（zab协议以保证数据的一致性）
  4. 更新请求顺序进行，来自同一个Client的更新请求按其发送顺序依次进行。
  5. 更新数据原子性：一次更新数据要么成功，要么失败。
  6. 实时性：在一定时间范围内，Client能读取到最新数据。

1.3、内存数据模型

ZooKeeper数据模型的结构与Unix文件系统很相似，整体上可以看做一棵树，每个节点称作为一个ZNode。每个ZNode默认能够存储1MB的数据，每个ZNode通过其路径唯一标识。

  • 每个子目录如/node1都被称作为一个znode(节点)。这个znode是被他所在的路径唯一标识。
  • znode可以有子节点目录，并且每个znode可以存储数据。
  • znode是有版本的，每个znode中存储的数据可以有多个版本，也就是一个访问路径可以存储多份数据。
  • znode可以被监控，包括这个目录节点中存储的数据的修改，子节点目录的变化等，一旦变化可以通知设置监控的客户端。

1.4、应用场景

提供的服务包括：统一命名服务，统一配置管理，统一集群管理，服务器节点动态上下线，软负载均衡。

2、节点的分类

2.1、持久节点

是指在节点创建后，就一直存在，直到有删除操作来主动删除这个节点—不会因为创建该节点的客户端会话失效而消失。

2.2、持久顺序节点

这类节点的基本特性和上面的节点类型是一致的，额外的特性是，在ZK中，每个父节点会为他的第一级节点维护一份时序，会记录每个子节点创建的先后顺序，基于这个特性，在创建子节点的时候，可以设置这个属性，那么在创建节点过程中，ZK会自动为节点名加一个数字后缀，作为新的节点名。这个数字后缀的范围是整型的最大值。

2.3、临时节点

和持久节点不同的是，临时节点的生命周期和客户端会话绑定，也就是说，客户端会话失效，那么这个节点就会自动被清除掉。注意，这里提到的是会话失效，而非连接断开，另外，在临时节点下面不能创建子节点。

2.4、临时顺序节点

具有临时节点特点，额外的特性是，每个父节点会为它的第一级子节点维护一份时序，这点和刚才提到的持久顺序节点类似。

3.5、配置文件

# 配置文件详解
tickTime =2000     #表示集群节点之间的心跳时间 2000毫秒 2秒
initLimit =10      #初始化集群时集群节点同步超时时间10*2(10个心跳) 20秒
syscLimit =5       #集群在运行过程中同步数据超时时间 10秒
dataDir            #默认数据存储位置
clientPort =2181   #代表Zookeeper服务监听的端口号
maxClientCnxns = 60 #最大连接数

4、Zookeeper客户端指令操作

ls path [watch]                  # 查看某个节点下的子节点  ls /zookeeper 加上ture则为监听
create [-s][-e] path data acl   # 创建节点[-s][-e]   create -s /node2 xiaoming
quit                              # 退出当前会话
stat path                       #查看节点状态 stat /node1
ls2 path                        #查看节点下的子节点和该节点的状态
set path data                   #修改节点数据
get path                        #获取节点上的数据    加上ture则为监听
history                         #展示历史记录
delete path                     #删除节点(不可删除含有子节点的节点)
rmr path                        #递归删除(删除该节点下的所有节点)

5、节点监听机制（Watch机制）

客户端可以监听znode节点的变化。znode节点节点的变化触发相应的事件，然后清除对该节点的监测。当监测一个znode节点时候，Zookeeper会发送通知给监测节点。一个Watch事件是一个一次性的监听器，当被设置了Watch的数据和目录发生了改变的时候，则服务器将这个改变发送给设置了Watch的客户端一边通知他们。

# ls path true    监听节点目录的变化
# get path true   监听节点数据的变化

6、Java操作Zookeeper

  • 导入依赖
<dependency>
    <groupId>com.101tec</groupId>
    <artifactId>zkclient</artifactId>
    <version>0.10</version>
</dependency>
  • 客户端连接代码
public class TestZKClient(){
    private ZkClient zkClient;


    //1.在zk创建节点
    public void testCreateNode(){
        //1.持久节点
        zkClient.create("/node","xiaochen",CreateMode.PERSISTENT);
        //2.持久顺序节点
        zkClient.create("/node","xiaochen",CreateMode.PERSISTENT_SEQUENTIAL);

        //3.临时节点
        zkClient.create("/node","xiaochen",CreateMode.EPHEMERAL_SEQUENTIAL);
        //4.临时顺序节点
        zkClient.create("/node","xiaochen",CreateMode.EPHEMERAL_SEQUENTIAL);
    } 


    //2.删除节点
    public void testDeleteNode(){
        //删除没有子节点的节点，返回值:是否删除成功
        boolean delete = zkClient.delete("/node");
        //递归删除节点信息，返回值:是否删除成功
        boolean recursive = zkClient.deleteRecursive("/node1");

    }

    //3.查询当前节点下的所有子节点
    public void testFindNodes(){
        //获取指定路径的节点信息，返回值:当前节点的子节点信息
        List<String> children = zkClient.getChildren("/");
        for(String child :children){
            System.out.println("child");
        }
    }
    //4.查看某个节点的数据 创建的节点数据应该与查询的节点数据序列化方式一致
    public void testFindNodeData(){
        Object readData = zkClient.readData("/node3");
        System.out.println(readData);
    }

    //5.查看节点状态信息
    public void testFindNodeDataAndStat(){
        Stat stat = new Stat();
        Object readData = zkClient.readData("/node",stat);
        System.out.println(readData);
        System.out.println(stat); 
    }

    //6.修改节点上的数据
    public void testWriteData(){
        User user = new User();
        user.setId(1);
        user.setName("小陈");
        zkClient.writeData("/node1",user);
    }

    //7.监听节点数据的变化
    public void testWatchDataChange() throws IOException{
        zkClient.subscribeDataChange("/node",new IZkDataListener(){
            //当前节点数据变化是触发这个方法
            public void handleDataChange(String dataPath,Object data)throws Exception{
            System.out.println("当前节点路径: " + dataPath);
            System.out.println("当前节点变化后数据: " +data);
            }
            //当前节点删除时触发这个方法
            public void handleDataDeleted(String dataPath) throws Exception{
                Syetem.out.println("当前节点路径: "+dataPath);
            }
        });
        Syetem.in.read();//阻塞当前监听，使其一直运行。
    }

    //8.监听节点目录变化
    public void testOnNodeChange() throws IOException{
        zkClient.subscribeChildChanges("/node1",new IZkChildListener(){
            //当节点的目录发生变化时，会自动调用这个方法
            //参数1:父节点名称
            //参数2:父节点中所有的子节点名称
            public void handleChildChange(String nodeName,List<String> list) throws Exception{
                System.out.println("父节点名称: " +nodeName);
                System.out.println("发生变更后子节点的名称: ");
                for(String name : list){
                    System.out.println(name);
                }
            }
        });
        Syetem.in.read();//阻塞当前监听，使其一直运行。
    }


    @Before  //初始化客户端对象
    public void before(){
        //参数1:zk server 服务器ip地址:端口号
        //参数2:会话超时时间
        //参数3:连接超时时间
        //参数4:序列化方式 对象
        zkClient = new ZkClient("10.15.0.5:2181",60000*30,60000,new SerializableSerializer());
    }

    @After //释放资源
    public void after(){
        zkClient.close();
    }
}

7、Zookeeper集群

7.1、集群

  1. 集群

集合同一种软件服务的多个节点同时提供服务

  1. 集群解决问题
  • 单节点的并发访问的压力问题
  • 单节点故障问题（硬件老化，自然灾害）

7.2、集群架构

7.3、集群搭建

至少三台

  • 创建Data目录
mkdir zkdata1 zkdata2 zkdata3
  • 在zkdata目录中创建myid文件
touch zkdata1/myid zkdata2/myid zkdata3/myid
  • 给myid文件添加数字
echo "1" >> zkdata1/myid
echo "2" >> zkdata2/myid
echo "3" >> zkdata3/myid
  • 在conf目录下创建三个zk配置文件，分别为zoo1.cfg,zoo2.cfg,zoo3.cfg

  • 启动zookeeper
./bin/zkServer.sh start /root/zkdata1/zoo.cfg
./bin/zkServer.sh start /root/zkdata2/zoo.cfg
./bin/zkServer.sh start /root/zkdata3/zoo.cfg

查看各个zk服务器的角色信息

./bin/zkServer.sh status /root/zkdata1/zoo.cfg
