封装JDBC

//封装数据库和数据连接
public class DBUtil{
    //封装数据库连接

    public static String driver = "com.mysql.jdbc.Driver";
    public static String db_url = "jdbc:mysql://localhost:3306/javaweb?serverTimezone=GMT%2B8"; 
    //db2为数据库名称
    public static String user = "root";
    public static String password="123456";

    public static Connection getConn(){
        Connection conn = null;

        try{
            Class.forName(driver);
            conn = DriverManager.getConnection(db_url,user,password);
        }catch(Exception ex){
            ex.printStackTrace();
        }
        return conn;
    }

    //关闭数据库连接,语句
    public static void close(Connection conn,Statement stmt){
        try{
            if(stmt != null){
                stmt.close();
            }
            if(conn != null){
                conn.close();
            }
        }catch(Exception ex){
            ex.printStackTrace();
        }
    }

    //关闭数据库连接,语句,结果集
    public static void close(Connection conn,Statement stmt,ResultSet rs){
        try{
            if(rs!=null){
                rs.close();
            }
            if(stmt != null){
                stmt.close();
            }
            if(conn != null){
                conn.close();
            }
        }catch(Exception ex){
            ex.printStackTrace();
        }
    }

}

DAO设计模式(同级目录分三层，建三个包)

  • Servlet（Controller）层

    调用Service层，接收结果。负责获取请求和处理响应，以及控制视图携带数据跳转（负责业务调度）

  • Service层

    调用Dao层，处理来自Dao的数据，写具体的业务处理。

  • Dao层

    通过dao层的操作可以从数据库中查询数据库并返回响应的数据

实体类的字段使用基本类型还是包装类?

/*
基本类型(int,long)
优点:
a.用于计算是效率高(effective java书上讲过的)
b.不会由于常量池引起比较大小错误(例如 java integer对象判断两个数字是否相等)

缺点:
1.当数据库中查询出结果之后封装结果集时如果返回的值时null时,如果直接赋值给这个基本类型的字段时,会在运行时报出异常,不能将null赋给一个基本类型,而用Integer就不存在这样都的情况,这个情况好像没有合适的避免办法.当没有默认值的情况下
2.当用spring的表单对象时如果页面传来的值是只有字段没有值是如果表单对象中有基本类型的值时会抛出异常.(可能存在,以前遇到过,记不清了)

缺点的规避方案:
设置在数据库中为该字段设置默认值,但是这经常会被忽略
表单对象不要缺少这个字段,缺少了之后会在request中换取到一个为null的值,当用这个字段没有在表单中被提交时就会报错.




包装类型(Integer,Long)
优点:
1.可以存放null,从数据库中查出值时可能会有null
2.表示一个值(不用于计算,只用于保存值时和int类型一样);

缺点:
不能用于两个Integer对象双等判断两个对象的值相等,会出现错误的.
不适合频繁(intger.maxvalue次左右)的用于计算,这个缺陷一般项目都遇不到的(猜测)

规避确定的方法:
要用双等判断两个对象的值是否相等时,要调用intValue方法
综上所述,在建立实体类时,用integer这个包装类比较好
*/

JavaBean和实体类的区别

实体类：

​ 须要有一些 private 的参数，以及针对参数的 setter 和 getter 方法来访问这些参数。java不包含业务逻辑或持久逻辑，不继承或不实现任何其它Java框架的类或接口。

JavaBean：

​ 首先明确的是JavaBean是一种Java类，并且是一种特殊的、可重用的类。java必须具备无参数的构造器，全部的属性都是private的，经过提供setter和getter方法来实现对成员属性的访问
　　JavaBean的种类按照功能能够划分为可视化和不可视化两类。可视化的JavaBean就是拥有GUI图形用户界面的，对最终用户是可见的。不可视化的JavaBean不要求继承，它更多的被使用在JSP中，一般状况下用来封装业务逻辑、数据分页逻辑、数据库操做和事物逻辑等，这样能够实现业务逻辑和前台程序的分离，提升了代码的可读性和易维护性，使系统更健壮和灵活。

编写流程

思路：

  • 先根据数据库表写好实体类
  • 根据要实现的功能写Dao层
  • 在Service层中调用Dao层，获取对应的方法，返回从Dao层中得到的结果集(没有结果集也要写，只是不用return)

  1. 编写字节码过滤器(解决汉字乱码问题)
/*
Java中的字符编码，默认为：ISO-8859-1（西方编码）。
这种编码方式为单字节码，只支持英文、俄文、葡萄牙文等乱七八糟文，不支持中文。
因此，我们需要对JSP页面或者Servlet中进行字符编码集的处理，国际编码（UTF-8）。
*/
//第一种解决办法,需要修改所有的Jsp和Servlet
//在JSP页面中把最上方标签中的字符编码ISO-8859-1改为UTF-8即可。
<%@ page language="java" import="java.util.*" pageEncoding="UTF-8"%>

//在Servlet中我们最常用的是设置下方代码进行解决乱码问题  
response.setCharacterEncoding("UTF-8");
//第二种解决办法,编写字符编码过滤器
//建立filter包,编写CharsetEncodingFilter
public class CharsetEncodingFilter implements Filter {  

    public void destroy() {

    }

    public void doFilter(ServletRequest request, ServletResponse response,
            FilterChain chain) throws IOException, ServletException {
        // 设置字符编码链锁
        request.setCharacterEncoding("utf-8");
        response.setCharacterEncoding("utf-8");
        chain.doFilter(request, response);

    }
    // 初始化
    public void init(FilterConfig config) throws ServletException {        
    }

}

  1. Dao层（负责数据处理）
public class UserDao {


    //查询user表全部数据
    public List<User> getAllUser() {
        Connection conn = null;
        Statement stmt = null;
        ResultSet rs = null;
        List<User> users = new ArrayList<>();
        try {

            conn = DBUtil.getConn();
            stmt = conn.createStatement();
            String sql = "select * from user";
            rs = stmt.executeQuery(sql);
            while (rs.next()) {
                String uId = rs.getString("uid");
                String uName = rs.getString("uname");

                //将遍历出来的值，set到user对象中
                User user = new User();
                user.setuId(uId);
                user.setuName(uName);

                //将对象user存入ArrayList中
                users.add(user);

            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            DBUtil.close(conn, stmt, rs);
        }
        return users;
    }


}
